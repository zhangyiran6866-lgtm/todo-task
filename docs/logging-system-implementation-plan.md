# 日志系统与日志中心实施计划

> **状态**：待实施  
> **关联阶段**：Phase 5（测试与上线）  
> **目标**：建立可检索、可追踪、可审计的日志体系，支撑问题排查和运维监控，并提供前端可视化入口。

---

## 一、设计目标

1. **排障效率**：每个请求可通过 `request_id` 串联入口日志、业务日志与错误日志。  
2. **运维可观测**：关键接口耗时、状态码、错误密度可快速检索。  
3. **安全合规**：敏感信息统一脱敏，日志访问受控。  
4. **前后端联动**：前端提供日志中心页面，支持筛选与详情查看。

---

## 二、范围定义

### 2.1 后端范围（Go + Gin + Zap）

- 统一日志字段模型：`timestamp`、`level`、`service`、`env`、`request_id`、`user_id`、`module`、`action`、`latency_ms`、`status_code`、`message`、`error`。
- 请求链路中间件：
  - 入站生成或透传 `X-Request-ID`。
  - 记录请求方法、路径、状态码、耗时、客户端 IP。
  - 将 `request_id` 注入 `context.Context`。
- 分级日志输出：
  - `app`：业务与访问日志。
  - `error`：错误与 panic 恢复日志。
  - `audit`：安全敏感行为（登录、登出、改密、删除任务）。
- 持久化策略：
  - 本地文件按天滚动（建议 `logs/app.log`、`logs/error.log`、`logs/audit.log`）。
  - 设置保留天数与压缩策略（配置化）。
- 查询能力：
  - 新增日志查询 API（仅授权用户可访问）。
  - 支持按级别、关键词、时间范围、模块、分页查询。

### 2.2 前端范围（Vue3 + TS）

- 入口位置：主导航新增“日志中心”入口（桌面与移动端均可达）。
- 新增页面：`LogCenterView.vue`。
- 功能能力：
  - 筛选项：级别、时间范围、关键词、模块。
  - 列表字段：时间、级别、模块、摘要、请求 ID。
  - 详情抽屉：完整日志 JSON、错误堆栈、复制 `request_id`。
  - 分页或滚动加载。
- 权限策略：
  - 无权限时不展示入口或页面内提示无权限。

---

## 三、接口草案（建议）

### 3.1 获取日志列表

- `GET /api/v1/logs`
- Query：
  - `level`: `debug|info|warn|error`
  - `module`: `auth|task|user|system`
  - `keyword`: 文本检索
  - `start_at` / `end_at`: ISO 时间
  - `limit` / `cursor`
- Response：
  - `items`: 日志摘要数组
  - `next_cursor`: 下一页游标

### 3.2 获取单条日志详情

- `GET /api/v1/logs/:id`
- Response：
  - 单条完整结构化日志对象（已脱敏）

## 四、实施步骤

1. **Step 1：日志规范落地**
   - 在 `pkg/logger` 扩展统一字段注入函数与日志格式约束。
   - 定义敏感字段脱敏工具（token/password/email）。

2. **Step 2：中间件与链路打通**
   - 新增请求日志中间件，挂载到 Gin。
   - 为 handler/service 传递 `request_id` 上下文能力。

3. **Step 3：持久化与保留策略**
   - 配置日志文件输出、按天切割、保留天数。
   - 在 `configs/config.yaml` 增加日志保留策略配置项。

4. **Step 4：日志查询 API**
   - 建立日志存储索引策略（文件检索或日志集合检索）。
   - 实现查询与详情接口，补充鉴权与限流。

5. **Step 5：前端日志中心**
   - 新增 API 层与页面视图。
   - 接入筛选、列表、详情抽屉与复制能力。
   - 导航入口联动（桌面+移动）。

6. **Step 6：联调与验收**
   - 人工造错验证链路：触发错误后按 `request_id` 完成全链路检索。
   - 校验脱敏与权限行为。

---

## 五、验收标准（DoD）

- [ ] 后端所有核心接口均带 `request_id` 且可追踪。  
- [ ] `app/error/audit` 三类日志可落盘并按策略保留。  
- [ ] 日志查询接口支持级别/关键词/时间范围过滤与分页。  
- [ ] 前端可通过“日志中心”查看日志列表与详情。  
- [ ] 敏感字段不以明文出现在日志输出与日志接口中。  
- [ ] 关键场景（登录失败、任务删除、500 错误）可通过日志完成问题定位。

---

## 六、风险与约束

1. **性能开销**：高频日志写入可能增加 I/O，需限制 debug 级别在生产启用。  
2. **存储增长**：日志体积增长快，必须启用过期清理与压缩。  
3. **安全风险**：若脱敏遗漏会造成敏感数据泄露，需加静态扫描与代码审查。  
4. **权限边界**：日志中心应仅对受控用户开放，避免越权查看。

---
